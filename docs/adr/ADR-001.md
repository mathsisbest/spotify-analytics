# ADR-001: BigQuery as the Analytical Data Warehouse

**Status:** Accepted

**Context:** The platform ingests Spotify listening history (tracks, timestamps, audio features) and needs to store, query, and transform this data for analytics and ML. We need a data warehouse that is serverless, scales to any volume, and integrates with Python and dbt.

**Decision:** Use Google BigQuery as the primary data warehouse.

Options considered:
- **PostgreSQL** — Requires provisioning, managing connections, and planning for storage growth. No native integration with dbt or Python ML tooling.
- **Snowflake** — Excellent but overkill for a single-user free-tier project. Higher complexity and cost.
- **BigQuery** — Serverless, free-tier eligible (1 TB/month query processing), native dbt adapter, native Python client, autoscaling, and tight GCP integration for the surrounding infrastructure.

**Consequences:**
- Zero server management — datasets, tables, and schemas are declarative via Terraform.
- Query costs are predictable and free-tier sufficient for personal use.
- Streaming inserts enable near-real-time ingestion from Cloud Functions.
- Vendor lock-in to GCP, but the dbt layer and SQL queries are portable.
