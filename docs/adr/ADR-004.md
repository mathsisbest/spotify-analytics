# ADR-004: Cloud Run for Serverless Execution

**Status:** Accepted

**Context:** The platform has several workloads that run periodically or on-demand: the ingestion pipeline (every 2 minutes), the Streamlit dashboard (continuously serving), and the ML training job (weekly). Each has different resource profiles and scheduling needs.

**Decision:** Use a combination of GCP serverless services — Cloud Functions (Gen2) for ingestion, Cloud Run (Service) for the dashboard, and Cloud Run (Job) for ML training.

Options considered:
- **Compute Engine / GKE** — Always-on and overkill for intermittent workloads. Higher cost and maintenance.
- **Cloud Functions for everything** — Good for short-lived ingestion but unsuitable for long-running ML training (timeout limits) or persistent web serving.
- **Cloud Run for everything** — Supports both services (web serving) and jobs (batch), scales to zero, and has a generous free tier.

**Consequences:**
- Three separate compute patterns: Pub/Sub-triggered function (ingestion), HTTP service (dashboard), scheduled job (ML).
- All three scale to zero — free tier covers personal use.
- Consistent deployment via Terraform + Docker.
- Cloud Scheduler orchestrates periodic execution via Pub/Sub (ingestion) and HTTP (ML job).
