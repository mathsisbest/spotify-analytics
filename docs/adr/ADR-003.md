# ADR-003: Streamlit for Interactive Dashboard

**Status:** Accepted

**Context:** The platform needs an interactive BI dashboard for exploring listening patterns, top artists/tracks, genre trends, and ML insights (forecast, mood map, recommendations). The dashboard must be lightweight, quick to build, and pair well with Plotly for custom visualizations.

**Decision:** Use Streamlit with Plotly for the dashboard.

Options considered:
- **Looker Studio** — Native BigQuery integration but limited custom theming and interactivity. Requires Google account for each viewer.
- **Grafana** — Excellent for time-series but awkward for the non-metric visualizations we need (genre area charts, mood map scatter plots).
- **Custom React app** — Full control but high build cost for a single-user dashboard.
- **Streamlit** — Python-native, works with Plotly out of the box, multi-page app support, simple caching, and free-tier deployable on Cloud Run.

**Consequences:**
- Dashboard is pure Python — shared types and utilities with the ingestion/ML codebase.
- `st.cache_data` provides memoization for BigQuery queries, reducing round trips.
- Multi-page structure via `pages/` directory enables file-disjoint parallel development.
- Cloud Run deployment is straightforward (single `Dockerfile`).
